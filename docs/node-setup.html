Complete step-by-step instructions for setting up and running your Bitcoin mainnet node

üéØ Goal: Set up a secure, efficient Bitcoin Core node that validates transactions and contributes to network security while minimizing storage requirements.

üöÄ Starting Your Node

Method 1: Using Built Bitcoin Core

# From build directory
cd build
./bin/bitcoind -prune=2000 -daemon

# Verify it's running
./bin/bitcoin-cli getblockchaininfo

Method 2: Official Bitcoin Core Binary

# If you downloaded official version
bitcoind -prune=2000 -daemon

# Check status
bitcoin-cli getblockchaininfo

Method 3: With Custom Configuration

# Using custom config file
./bin/bitcoind -conf=/path/to/bitcoin.conf -daemon

# Or specify data directory
./bin/bitcoind -datadir=/custom/path -prune=2000 -daemon

üìä Monitoring Your Node

Check Blockchain Sync Status

# Get comprehensive blockchain info
./bin/bitcoin-cli getblockchaininfo

# Quick sync progress check
./bin/bitcoin-cli getblockchaininfo | grep -E "(blocks|headers|verificationprogress)"

# Check if initial download is complete
./bin/bitcoin-cli getblockchaininfo | grep "initialblockdownload"

Monitor Network Connections

# Check number of connected peers
./bin/bitcoin-cli getconnectioncount

# Detailed peer information
./bin/bitcoin-cli getpeerinfo

# Network status
./bin/bitcoin-cli getnetworkinfo

Verify Pruning Status

# Check pruning configuration
./bin/bitcoin-cli getblockchaininfo | grep -E "(pruned|pruneheight|prune_target_size)"

# Monitor disk usage
du -sh ~/Library/Application\ Support/Bitcoin/

üîß Essential Node Management Commands

Start/Stop Operations

# Start node in background
./bin/bitcoind -prune=2000 -daemon

# Stop node gracefully
./bin/bitcoin-cli stop

# Check if node is running
./bin/bitcoin-cli getblockchaininfo > /dev/null && echo "Node is running" || echo "Node is stopped"

Log Monitoring

# Monitor live log output
tail -f ~/Library/Application\ Support/Bitcoin/debug.log

# Search for errors
grep -i error ~/Library/Application\ Support/Bitcoin/debug.log

# Filter by category
grep "net:" ~/Library/Application\ Support/Bitcoin/debug.log

Emergency Recovery

üíº Wallet Management

# Get wallet information

# Generate a receiving address

# Check wallet balance

Wallet Security

üîí Security Note: Never use mainnet wallets with test builds or share private keys 
# Create encrypted wallet

# Backup wallet
./bin/bitcoin-cli backupwallet "/safe/location/wallet_backup.dat"

# Load existing wallet
./bin/bitcoin-cli loadwallet "wallet_name"

üéØ Success Indicators

‚úÖ Your node is working correctly when:

  • verificationprogress approaches 1.0 (99.9%+)
  • "initialblockdownload": false in blockchain info
  • "pruned": true confirms pruning is active
  • Connected to 8+ peers consistently
  • Data directory size stable around your prune target
  • Can create wallets and generate addresses

üîç Troubleshooting Common Issues

Sync Problems

Slow Sync Speed

  • Check internet connection stability
  • Ensure sufficient disk space available
  • Try increasing -dbcache parameter
  • Consider using -assumevalid for faster initial sync

Connection Issues

  • Check firewall settings (port 8333)
  • Verify network connectivity
  • Try different DNS servers
  • Use -addnode to connect to specific peers

Resource Issues

High Memory Usage

  • Reduce -dbcache value (default: auto)
  • Lower -maxmempool size
  • Increase prune value to reduce I/O

‚öôÔ∏è Optimization Tips

Performance Tuning

# Optimized startup for faster sync
./bin/bitcoind -prune=2000 -daemon \
  -dbcache=1000 \
  -maxconnections=16 \
  -par=4 \
  -checkblocks=24

# Memory-constrained setup
./bin/bitcoind -prune=550 -daemon \
  -dbcache=50 \
  -maxconnections=8 \
  -maxmempool=50

Network Optimization

# Limit bandwidth usage
./bin/bitcoind -prune=2000 -daemon \
  -maxuploadtarget=1000 \
  -maxconnections=8

# For unstable connections
./bin/bitcoind -prune=2000 -daemon \
  -timeout=60000 \
  -peertimeout=120
